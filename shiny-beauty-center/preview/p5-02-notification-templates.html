Architecture Notes

Flag-off safety

When NEXT_PUBLIC_NOTIFICATIONS_ENABLED=false:

  • All send calls return immediately with {ok: true, externalId: null}
  • A single console.log line is emitted
  • No SDK is imported or instantiated
  • Booking flows are never blocked

Dedup approach

Cron route uses the notifications table as a sent-log:

  • Inserts a sentinel row with subject __reminder_24h__:<appt_id>
  • If row already exists → skip (dedup)
  • Postgres unique constraint on (user_id, subject) prevents race conditions
  • Cron runs every 15 min; windows are ±8 min around 24h/2h thresholds

Providers

  • Email: Resend (primary) or SMTP fallback
  • WhatsApp: WATI (Gulf-friendly BSP)
  • Both lazily constructed — no crash on missing keys

Triggers

  • Pay-at-salon: confirmation fires in createAppointmentAction
  • Online payment: confirmation fires in tap-webhook/route.ts
  • Both use sendBookingConfirmation()
  • Reminders: GET /api/cron/reminders via Vercel cron
